# 11. sparse 검색 vs embedding 검색

이 문서는 수강생이 가장 헷갈려하는 부분을 쉽게 비교하기 위한 자료다.

## 한 줄 요약

- sparse 검색 = **단어가 얼마나 겹치는가**를 본다
- embedding 검색 = **의미가 얼마나 비슷한가**를 본다

## 예시 질문

질문:
- 고객들이 가격에 대해 많이 물어보나요?

문서 표현:
- 도입 비용 문의가 반복적으로 등장했다
- 예산 관련 질문이 많았다

### sparse 검색 관점
"가격"이라는 단어가 직접 없으면 놓칠 수 있다.

### embedding 검색 관점
"가격", "비용", "예산"이 비슷한 뜻이라는 점을 반영해서 더 잘 찾을 수 있다.

## 언제 sparse가 좋은가?

- 개념 설명용 입문 수업
- 작은 데이터셋
- 빠르게 구조를 보여주고 싶을 때
- 인프라를 단순하게 유지하고 싶을 때

## 언제 embedding이 좋은가?

- 표현이 다양한 실제 문서
- 검색 품질이 중요한 서비스
- 사내 문서, 매뉴얼, 상담 기록처럼 문장이 다양한 경우
- 벡터DB까지 확장할 계획이 있는 경우

## 강의에서 추천하는 순서

1. sparse 검색으로 먼저 이해시킨다
2. 같은 질문으로 embedding 검색도 보여준다
3. 왜 결과가 달라지는지 비교한다
4. 그 다음 벡터DB와 임베딩 모델 이야기를 붙인다

## 수강생에게 남겨야 할 핵심

좋은 RAG는 결국 **잘 찾는 능력**에서 시작한다.
그리고 임베딩은 그 "잘 찾기"를 한 단계 더 발전시키는 기술이다.
