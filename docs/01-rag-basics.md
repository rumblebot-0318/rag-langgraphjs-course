# 01. RAG 기초

## RAG란?

RAG는 Retrieval-Augmented Generation의 약자다.
쉽게 말하면 **먼저 찾고, 그 다음 답하는 방식**이다.

## 왜 필요한가?

LLM만 단독으로 쓰면:
- 최신 정보가 부족할 수 있고
- 내부 문서를 모를 수 있고
- 환각이 생길 수 있다

RAG는 이 문제를 줄이기 위해 외부 지식을 가져와서 답변에 붙인다.

## 기본 흐름

1. 질문 받기
2. 관련 문서 검색
3. 검색된 문서를 컨텍스트로 정리
4. 모델이 답변 생성

## 수업에서 강조할 포인트

- 좋은 답변은 좋은 검색에서 시작한다
- RAG는 모델만의 문제가 아니라 데이터 파이프라인 문제다
- 문서 품질, chunking, 검색 전략이 매우 중요하다
