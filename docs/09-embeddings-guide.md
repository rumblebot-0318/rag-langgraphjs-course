# 09. 임베딩 가이드

## 임베딩이란?

임베딩은 텍스트를 숫자 벡터(vector)로 바꾸는 방식이다.
쉽게 말하면 문장의 의미를 좌표처럼 표현하는 것이다.

예를 들어 아래 문장들은 단어가 조금 달라도 의미가 비슷하다.
- 가격이 얼마인가요?
- 비용이 어떻게 되나요?
- 도입 금액이 궁금합니다.

키워드 검색만 하면 단어가 다를 때 놓칠 수 있다.
하지만 임베딩을 쓰면 **의미가 비슷한 문장끼리 가까운 벡터**로 표현할 수 있다.

## 왜 쓰는가?

RAG에서 검색 품질은 매우 중요하다.
임베딩을 쓰는 이유는 보통 다음과 같다.

1. **의미 기반 검색**이 가능하다
2. 같은 뜻의 다른 표현을 더 잘 잡을 수 있다
3. 키워드가 정확히 일치하지 않아도 관련 문서를 찾기 쉽다
4. 문서 검색 품질이 올라가면 최종 답변 품질도 좋아진다

## sparse 검색과 차이

### sparse 검색
- 단어 겹침 중심
- 빠르고 단순함
- 입문용 설명에 좋음
- 그러나 표현이 조금만 달라도 놓칠 수 있음

### embedding 검색
- 의미 유사도 중심
- 더 자연스러운 검색 가능
- 보통 벡터DB와 같이 많이 씀
- 다만 모델과 인프라 비용이 늘 수 있음

## 강의 포인트

- 처음엔 sparse 검색으로 원리를 설명하고
- 그 다음 임베딩을 붙이면 "왜 검색 품질이 좋아지는지" 체감시키기 좋다
- 즉, 임베딩은 RAG를 더 실무적으로 만들어주는 핵심 부품 중 하나다
